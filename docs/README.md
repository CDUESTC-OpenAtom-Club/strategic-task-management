# 前端架构修复方案 - 文档总览

## 概述

本目录包含完整的前端架构修复方案，用于将当前项目迁移到标准的 6 层架构。

## 方案特点

- **模块化工作分配**：8 个 Agent 并行协作
- **分阶段实施**：8 个清晰的实施阶段
- **完整的备份与回滚策略**：确保代码安全
- **全面的测试策略**：单元测试 + 集成测试 + 基于属性的测试
- **详细的迁移映射**：每个文件的目标位置都有明确定义

## 当前项目分析

通过代码分析，发现：

✅ **项目已有良好基础：**
- `features/` 目录按业务领域划分，结构相对规范
- `pages/` 目录按路由组织清晰
- `shared/` 目录包含共享 UI 组件
- `entities/` 目录已存在
- 已有完整的测试基础设施（Vitest + Vue Test Utils）
- 已有基于属性的测试

❌ **需要调整的部分：**
- 缺少数字前缀的目录命名规范
- 顶级文件（App.vue、main.ts）位置需要调整
- `composables/` 需要移到 `5-shared/lib/hooks/`
- `utils/` 需要移到 `5-shared/lib/utils/`
- `api/` 需要移到 `5-shared/lib/api/`
- 测试目录结构需要与源代码匹配

1. 先阅读 **architecture-refactor-plan.md** - 了解整体计划

## 关键注意事项

### 备份与回滚
- 在开始前务必创建完整备份
- 每个阶段完成后提交代码
- 保留原始代码副本以便快速回滚

### 增量迁移
- 分阶段实施，每阶段完成后测试
- 不要一次性迁移所有文件
- 使用 git mv 保留文件历史

### 测试优先
- 确保现有测试通过
- 每个阶段完成后运行测试
- 重构过程中补充缺失的测试

### 导入路径更新
- 使用 IDE 的全局搜索替换功能
- 更新 vite.config.ts 和 tsconfig.json 的路径别名
- 检查所有 import 语句

## 风险评估

| 风险项 | 影响 | 概率 | 缓解措施 |
|--------|------|------|---------|
| 导入路径错误导致构建失败 | 高 | 中 | 分阶段验证，每阶段后立即构建 |
| 测试失败未及时发现 | 高 | 中 | 每阶段完成后运行完整测试 |
| 路由配置错误导致页面无法访问 | 中 | 低 | 完整测试所有路由 |
| 类型检查失败 | 高 | 高 | 使用 TypeScript 严格模式，逐步验证 |
