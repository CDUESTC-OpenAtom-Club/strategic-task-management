## 功能说明

### WebSocket连接流程

1. **用户登录** → 触发 `onMounted`
2. **初始化通知** → 调用 `initApprovalNotifications()`
3. **连接WebSocket** → 连接到 `ws://host/ws/notifications?userId={userId}`
4. **接收消息** → 解析并显示通知
5. **自动重连** → 连接断开时自动重试（最多5次）

### 通知类型

| 类型               | 图标 | Element Plus类型 | 说明       |
| ------------------ | ---- | ---------------- | ---------- |
| APPROVAL_REQUIRED  | 📋   | warning          | 需要审批   |
| APPROVAL_APPROVED  | ✅   | success          | 审批通过   |
| APPROVAL_REJECTED  | ❌   | error            | 审批拒绝   |
| APPROVAL_NEXT_STEP | ➡️   | info             | 进入下一步 |
| SYSTEM             | 📢   | info             | 系统通知   |

### 通知显示

**Element Plus通知**:

- 位置: 右上角
- 持续时间: 5秒
- 可点击跳转到审批页面

**浏览器通知**:

- 需要用户授权
- 应用在后台时也能收到
- 显示标题和内容

## 安全性考虑

1. **用户ID验证**: 从localStorage获取，确保用户已登录
2. **连接清理**: 退出登录时断开连接
3. **重连限制**: 最多重试5次，防止无限重连
4. **错误处理**: 捕获并记录所有错误

## 性能优化

1. **按需连接**: 只在用户登录后连接
2. **自动清理**: 应用卸载时断开连接
3. **消息队列**: 使用数组存储通知，支持批量处理
4. **事件分发**: 使用CustomEvent分发通知，解耦组件
