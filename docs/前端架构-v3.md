## 核心优化策略

### 1. 代码分割 (Code Splitting)

#### 1.1 路由级代码分割

所有路由页面采用动态导入，实现按需加载：

```typescript
// src/router/index.ts
const routes = [
  {
    path: '/dashboard',
    component: () => import('@/pages/dashboard/ui/DashboardPage.vue')
  },
  {
    path: '/plans',
    component: () => import('@/pages/strategy/plans/ui/PlanListPage.vue')
  }
  // ... 其他路由
]
```

**优势**:

- 减少初始包体积
- 提升首屏加载速度
- 按需加载页面代码

#### 1.2 组件级代码分割

大型组件使用 `defineAsyncComponent` 懒加载：

```typescript
import { defineAsyncComponent } from 'vue'

// ECharts 图表组件懒加载
const ScoreCompositionChart = defineAsyncComponent(
  () => import('@/shared/ui/charts/ScoreCompositionChart.vue')
)

const AlertDistributionChart = defineAsyncComponent(
  () => import('@/shared/ui/charts/AlertDistributionChart.vue')
)
```

**工具函数**: `src/utils/asyncComponent.ts`

```typescript
import { createAsyncComponent } from '@/utils/asyncComponent'

const HeavyComponent = createAsyncComponent(() => import('@/components/HeavyComponent.vue'), {
  delay: 200, // 延迟显示 loading
  timeout: 10000, // 超时时间
  maxRetries: 2, // 重试次数
  showSkeleton: true // 显示骨架屏
})
```

#### 1.3 Vite 构建优化

**vite.config.ts** 配置的代码分割策略：

```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks(id) {
        // Vue 核心库
        if (id.includes('vue') || id.includes('pinia')) {
          return 'vue-core'
        }
        // Vue Router
        if (id.includes('vue-router')) {
          return 'vue-router'
        }
        // Element Plus UI 框架
        if (id.includes('element-plus')) {
          return 'element-plus'
        }
        // ECharts 图表库
        if (id.includes('echarts')) {
          return 'echarts'
        }
        // 业务功能模块
        if (id.includes('/src/features/dashboard/')) {
          return 'feature-dashboard'
        }
        // ... 更多分组
      }
    }
  }
}
```

**生成的 Chunk 分组**:

- `vue-core.js` - Vue 核心库
- `vue-router.js` - 路由库
- `element-plus.js` - UI 组件库
- `echarts.js` - 图表库
- `xlsx.js` - Excel 处理
- `pdf.js` - PDF 生成
- `feature-*.js` - 业务功能模块
- `shared-ui.js` - 共享 UI 组件

### 2. 资源优化

#### 2.1 Tree Shaking

- **Element Plus**: 自动导入，按需加载
- **lodash-es**: 使用 ES Module 版本
- **ECharts**: 按需引入图表类型

```typescript
// vite.config.ts
AutoImport({
  resolvers: [ElementPlusResolver()]
})
```

- 使用 `font-display: swap`

#### 3.2 LocalStorage 缓存

- 用户信息

#### 3.3 Service Worker（规划中）

- 离线支持

### 4. 性能监控

#### 4.1 组件加载性能

```typescript
// src/utils/asyncComponent.ts
const startTime = performance.now()
const component = await loader()
const loadTime = performance.now() - startTime

if (loadTime > 1000) {
  logger.warn(`组件加载耗时: ${loadTime.toFixed(2)}ms`)
}
```

#### 4.2 路由切换性能

```typescript
// src/router/nprogress.ts
router.beforeEach(() => {
  startProgress() // 显示进度条
})

router.afterEach(() => {
  doneProgress() // 隐藏进度条
})
```

#### 4.3 API 请求性能

```typescript
// src/shared/api/interceptors/performanceInterceptor.ts
interceptors.request.use(config => {
  config.metadata = { startTime: performance.now() }
  return config
})

interceptors.response.use(response => {
  const duration = performance.now() - response.config.metadata.startTime
  if (duration > 3000) {
    logger.warn(`API 请求耗时: ${duration.toFixed(2)}ms`)
  }
  return response
})
```

## 最佳实践

### 1. 组件懒加载原则

**需要懒加载的组件**:

- ECharts 图表组件
- 大型表单组件
- 弹窗/对话框组件
- 条件渲染的复杂组件

**不需要懒加载的组件**:

- 基础 UI 组件（按钮、输入框等）
- 小型工具组件
- 频繁使用的组件

### 2. 代码分割原则

**按业务模块分割**:

- 每个独立的业务功能模块一个 chunk
- 共享的 UI 组件单独一个 chunk
- 第三方库按使用频率分组

**避免过度分割**:

- 单个 chunk 不小于 10KB
- 避免过多 HTTP 请求

### 3. 性能优化检查清单

- [ ] 所有路由使用动态导入
- [ ] 大型组件使用异步加载
