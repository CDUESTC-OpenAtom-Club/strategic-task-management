## 前端实现

### 核心组件

#### 1. PlanApprovalDrawer.vue

**路径**: `src/features/approval/components/PlanApprovalDrawer.vue`

**功能**: 计划审批抽屉组件

**Props**:

```typescript
interface Props {
  visible: boolean // 抽屉可见性
}
```

**Emits**:

```typescript
interface Emits {
  (e: 'update:visible', value: boolean): void
  (e: 'close'): void
  (e: 'refresh'): void
}
```

**核心方法**:

- `loadPendingApprovals()`: 加载待审批列表
- `handleApprove()`: 审批通过
- `handleReject()`: 审批拒绝

#### 2. WebSocket服务

**路径**: `src/shared/services/websocket.ts`

**功能**: WebSocket连接管理和消息处理

**核心方法**:

```typescript
// 初始化连接
function initWebSocket(userId: number): void

// 断开连接
function disconnectWebSocket(): void

// 发送消息
function sendMessage(message: any): void
```

**连接URL**: `ws://host/ws/notifications?userId={userId}`

**重连机制**:

- 最多重试5次
- 间隔5秒
- 指数退避

#### 3. 审批通知服务

**路径**: `src/features/approval/services/approvalNotifications.ts`

**功能**: 处理审批通知并显示

**通知类型**:

```typescript
type NotificationType =
  | 'APPROVAL_REQUIRED' // 需要审批
  | 'APPROVAL_APPROVED' // 审批通过
  | 'APPROVAL_REJECTED' // 审批拒绝
  | 'APPROVAL_NEXT_STEP' // 进入下一步
  | 'SYSTEM' // 系统通知
```

**显示方式**:

- Element Plus通知（应用内）
- 浏览器通知（后台）

## 后端实现

### Controller层

**类名**: `PlanApprovalController`

**路径**: `src/main/java/com/sism/controller/PlanApprovalController.java`

**端点列表**:

```java
@RestController
@RequestMapping("/api/plans/approval")
public class PlanApprovalController {

    // 提交计划审批
    @PostMapping("/{planId}/submit")
    ResponseEntity<ApiResponse<PlanVO>> submitPlanForApproval(
        @PathVariable Long planId,
        @RequestParam Long userId
    )

    // 审批通过
    @PostMapping("/instances/{instanceId}/approve")
    ResponseEntity<ApiResponse<String>> approvePlan(
        @PathVariable Long instanceId,
        @RequestBody Map<String, Object> request
    )

    // 审批拒绝
    @PostMapping("/instances/{instanceId}/reject")
    ResponseEntity<ApiResponse<String>> rejectPlan(
        @PathVariable Long instanceId,
        @RequestBody Map<String, Object> request
    )

    // 获取待审批列表
    @GetMapping("/pending")
    ResponseEntity<ApiResponse<List<AuditInstance>>> getPendingApprovals(
        @RequestParam Long userId
    )

    // 获取审批状态
    @GetMapping("/plans/{planId}/status")
    ResponseEntity<ApiResponse<AuditInstance>> getPlanApprovalStatus(
        @PathVariable Long planId
    )

    // 统计待审批数量
    @GetMapping("/pending/count")
    ResponseEntity<ApiResponse<Long>> countPendingApprovals(
        @RequestParam Long userId
    )

    // 获取当前步骤
    @GetMapping("/instances/{instanceId}/current-step")
    ResponseEntity<ApiResponse<String>> getCurrentStep(
        @PathVariable Long instanceId
    )
}
```

### Service层

#### 1. PlanService

**方法**: `submitPlanForApproval(Long planId, Long userId)`

**功能**: 提交计划进行审批

**流程**:

1. 验证计划状态（必须是DRAFT）
2. 选择审批流程（STRATEGY/FUNCDEPT）
3. 创建审批实例
4. 发送通知给一级主管

#### 2. ApprovalService

**方法**: `approvePlan(Long instanceId, Long approverId, String comment)`

**功能**: 审批通过

**流程**:

1. 验证审批人权限
2. 写入审批记录
3. 推进到下一步骤
4. 如果是终审，更新业务状态

**方法**: `rejectPlan(Long instanceId, Long approverId, String comment)`

**功能**: 审批拒绝

**流程**:

1. 验证审批人权限
2. 写入审批记录
3. 终止审批流程
4. 更新业务状态为REJECTED

#### 3. AuditInstanceService

**功能**: 审批实例管理

**核心方法**:

- `createAuditInstance()`: 创建审批实例
- `advanceToNextStep()`: 推进到下一步
- `terminateInstance()`: 终止审批
- `getCurrentStepDescription()`: 获取当前步骤描述

## 数据库设计

### 核心表结构

#### 1. audit_flow_def (审批流程定义)

```sql
CREATE TABLE audit_flow_def (
    flow_id BIGSERIAL PRIMARY KEY,
    flow_code VARCHAR(50) NOT NULL UNIQUE,
    flow_name VARCHAR(100) NOT NULL,
    entity_type VARCHAR(50) NOT NULL,
    description TEXT,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**示例数据**:

- `PLAN_DISPATCH_STRATEGY`: 战略发展部计划审批流程
- `PLAN_DISPATCH_FUNCDEPT`: 职能部门计划审批流程

#### 2. audit_step_def (审批步骤定义)

```sql
CREATE TABLE audit_step_def (
    step_id BIGSERIAL PRIMARY KEY,
    flow_id BIGINT NOT NULL,
    step_code VARCHAR(50) NOT NULL,
    step_name VARCHAR(100) NOT NULL,
    step_order INT NOT NULL,
    approver_role VARCHAR(50),
    is_terminal BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**步骤顺序**:

1. step10: 填报人
2. step20: 一级主管
3. step30: 二级主管
4. step40: 上级部门
5. step50: 战略发展部

#### 3. audit_instance (审批实例)

```sql
CREATE TABLE audit_instance (
    instance_id BIGSERIAL PRIMARY KEY,
    flow_id BIGINT NOT NULL,
    entity_type VARCHAR(50) NOT NULL,
    entity_id BIGINT NOT NULL,
    current_step_id BIGINT,
    status VARCHAR(20) NOT NULL,
    submitter_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    finished_at TIMESTAMP
);
```

**状态枚举**:

- `PENDING`: 审批中
- `APPROVED`: 已通过
- `REJECTED`: 已拒绝
- `CANCELLED`: 已取消

#### 4. approval_record (审批记录)

```sql
CREATE TABLE approval_record (
    record_id BIGSERIAL PRIMARY KEY,
    instance_id BIGINT NOT NULL,
    step_id BIGINT NOT NULL,
    approver_id BIGINT NOT NULL,
    action VARCHAR(20) NOT NULL,
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**操作类型**:

- `APPROVE`: 通过
- `REJECT`: 拒绝
- `DELEGATE`: 委托

## 故障排查

### 常见问题

#### 1. WebSocket连接失败

**症状**: 无法收到实时通知

**排查**:

```bash
# 检查WebSocket端点
curl -i -N -H "Connection: Upgrade" \
  -H "Upgrade: websocket" \
  http://localhost:8080/ws/notifications
```

**解决**: 检查防火墙、代理配置

#### 2. 审批提交失败

**症状**: 点击"提交审批"无响应

**排查**:

1. 检查浏览器控制台错误
2. 检查网络请求状态
3. 检查后端日志

**解决**: 根据错误信息修复

#### 3. 权重验证失败

**症状**: 提示"权重总和必须为100%"

**排查**:

```sql
SELECT SUM(weight_percent) FROM indicator
WHERE year = 2025 AND owner_dept = '战略发展部';
```

**解决**: 调整指标权重
