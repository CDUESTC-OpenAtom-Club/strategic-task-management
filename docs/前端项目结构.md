## 项目概述

本项目是一个基于Vue 3 + TypeScript的战略任务管理系统前端，采用现代化的前端架构设计。系统提供用户管理、组织管理、战略规划、指标管理、任务跟踪、进度报告、审批流程、通知告警等核心功能的Web界面。

### 核心特性

- ✅ **现代化技术栈**: Vue 3 + TypeScript + Vite
- ✅ **组件化开发**: Element Plus UI组件库
- ✅ **状态管理**: Pinia状态管理
- ✅ **路由管理**: Vue Router 4
- ✅ **数据可视化**: ECharts图表库
- ✅ **类型安全**: TypeScript + Zod数据验证
- ✅ **API封装**: Axios拦截器和重试机制
- ✅ **代码规范**: ESLint + Prettier + Husky
- ✅ **测试覆盖**: Vitest单元测试

## 技术栈

### 核心框架

- **Vue 3.5** - 渐进式JavaScript框架
- **TypeScript 5.9** - JavaScript超集
- **Vite 7** - 新一代前端构建工具
- **Vue Router 4.5** - 官方路由管理器

### UI组件库

- **Element Plus 2.12** - 基于Vue 3的组件库
- **@element-plus/icons-vue** - Element Plus图标库

### 状态管理和数据

- **Pinia 2.2** - Vue官方状态管理库
- **VueUse** - Vue组合式API工具集
- **dayjs** - 轻量级日期处理库
- **lodash-es** - JavaScript实用工具库

### 数据可视化

- **ECharts 5.5** - 强大的数据可视化库
- **vue-echarts 7** - ECharts的Vue组件封装

### HTTP客户端

- **Axios 1.7** - HTTP客户端
- **Zod 4.3** - TypeScript优先的模式验证库

### 文档导出

- **html2canvas** - HTML转Canvas
- **jspdf** - PDF生成库
- **xlsx** - Excel文件处理库

### 开发工具

- **ESLint 8** - 代码检查工具
- **Prettier 3** - 代码格式化工具
- **Husky 9** - Git钩子工具
- **lint-staged** - Git暂存文件检查
- **Vitest 4** - 单元测试框架

### 路由守卫

```typescript
router.beforeEach(async (to, from, next) => {
  // 1. 检查是否需要认证
  const requiresAuth = to.matched.some(record => record.meta.requiresAuth)

  if (requiresAuth) {
    // 2. 检查用户是否登录
    const authStore = useAuthStore()
    if (!authStore.isLoggedIn) {
      // 未登录，跳转到登录页
      next({ name: 'Login', query: { redirect: to.fullPath } })
      return
    }

    // 3. 检查权限
    const permission = to.meta.permission
    if (permission && !authStore.hasPermission(permission)) {
      // 无权限，跳转到403页面
      next({ name: 'Forbidden' })
      return
    }
  }

  // 4. 设置页面加载进度
  NProgress.start()

  next()
})

router.afterEach(() => {
  NProgress.done()
})
```

## 开发指南

### 添加新功能的步骤

1. **创建API接口**: 在`src/api/`中添加API函数
2. **创建Service**: 在`src/services/`中创建服务层
3. **创建Store**: 在`src/stores/`中创建状态管理
4. **创建视图组件**: 在`src/views/`或`src/features/`中创建视图
5. **配置路由**: 在`src/router/`中添加路由
6. **创建组件**: 在`src/components/`中创建可复用组件
7. **编写测试**: 在相应的`__tests__`目录编写测试
8. **更新文档**: 更新相关文档

### 代码规范

**组件命名**:

- 使用PascalCase命名组件文件
- 组件名使用多个单词，避免与HTML标签冲突

**目录结构**:

- 按功能模块划分目录
- 相关文件放在同一目录下
- 使用index.ts统一导出

**TypeScript使用**:

- 定义明确的接口和类型
- 避免使用any类型
- 使用类型推导

**样式编写**:

- 使用SCSS预处理器
- 使用BEM命名规范
- 避免深层嵌套

**性能优化**:

- 使用路由懒加载
- 合理使用computed和watch
- 避免不必要的响应式数据

### 测试策略

**单元测试**:

- 使用Vitest进行单元测试
- 测试覆盖率目标：80%+
- 关键业务逻辑必须有测试

**组件测试**:

- 使用@vue/test-utils测试组件
- 测试组件的渲染和交互

**E2E测试**:

- 测试关键业务流程
- 测试用户交互场景
