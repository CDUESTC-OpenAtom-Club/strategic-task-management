# 前端架构修复方案

## 概述

当前项目架构与目标架构存在差异，需要进行全面重构以符合 `src/` 目录下的架构规范。本方案将项目重构分为多个阶段，每个阶段由专门的 Agent 负责。

## 目标架构

```
/src
├── 1-app/                 # 应用入口与核心配置
├── 2-pages/               # 页面组件
├── 3-features/            # 业务功能模块
├── 4-entities/            # 核心业务实体
├── 5-shared/              # 共享资源
└── 6-processes/           # 跨功能流程
```

## 当前架构分析

通过分析项目结构，发现以下问题：

1. **目录结构不规范**：当前使用 `src/App.vue`、`src/main.ts` 等顶级文件，而非 `1-app/` 目录
2. **功能模块分散**：features 目录结构不清晰，缺少明确的子目录划分
3. **共享资源未集中**：工具函数、UI 组件、配置等未统一放置在 `5-shared/` 目录
4. **缺少 entities 层**：核心业务实体未单独抽象
5. **测试结构不完整**：当前 tests 目录结构与源代码结构不匹配

---

## 阶段一：准备与分析 (Agent 1 - 架构分析师)

### 任务：
1. 完整扫描当前项目结构
2. 分析每个文件的功能与所属模块
3. 创建详细的文件迁移映射表
4. 识别架构重构风险点
5. 制定备份与回滚策略

### 输出：
- 项目结构扫描报告
- 文件迁移映射表
- 架构风险评估报告
- 备份与回滚方案

---

## 阶段二：架构基础重构 (Agent 2 - 架构重构师)

### 任务：
1. 创建目标架构目录结构
2. 迁移应用入口文件到 `1-app/` 目录
3. 重构路由、状态管理、错误处理等核心配置
4. 建立全局样式与资源管理体系
5. 更新 vite 与 vitest 配置

### 输出：
- 规范的项目目录结构
- 重构后的核心配置文件
- 更新的构建与测试配置

---

## 阶段三：页面与路由重构 (Agent 3 - 页面重构师)

### 任务：
1. 分析现有页面组件
2. 创建 `2-pages/` 目录结构
3. 迁移页面组件到对应目录
4. 重构路由配置
5. 确保页面间导航正常

### 输出：
- 规范的页面目录结构
- 更新的路由配置
- 可正常访问的所有页面

---

## 阶段四：功能模块重构 (Agent 4 - 功能模块重构师)

### 任务：
1. 分析现有 features 目录
2. 重构每个业务功能模块
3. 确保每个 feature 包含 ui/model/api/lib 子目录
4. 分离业务逻辑与 UI 组件
5. 建立功能模块间的通信机制

### 输出：
- 规范的功能模块结构
- 清晰的业务逻辑分层
- 可复用的功能组件

---

## 阶段五：实体层与共享资源重构 (Agent 5 - 架构优化师)

### 任务：
1. 抽象核心业务实体到 `4-entities/` 目录
2. 集中管理共享 UI 组件到 `5-shared/ui/`
3. 重构共享工具函数到 `5-shared/lib/`
4. 建立类型定义与常量的集中管理
5. 创建 API 客户端与错误处理的统一接口

### 输出：
- 完整的实体层抽象
- 统一的共享资源库
- 规范的 API 通信机制

---

## 阶段六：流程与集成重构 (Agent 6 - 流程重构师)

### 任务：
1. 识别跨功能流程
2. 创建 `6-processes/` 目录
3. 重构审批流程、数据导出等复杂流程
4. 确保流程间的协作与集成
5. 优化用户交互流程

### 输出：
- 清晰的流程定义
- 优化的用户交互
- 稳定的流程集成

---

## 阶段七：测试重构 (Agent 7 - 测试架构师)

### 任务：
1. 分析现有测试结构
2. 创建与源代码结构匹配的测试目录
3. 重构单元测试
4. 增加集成测试覆盖
5. 创建 E2E 测试框架
6. 确保所有测试通过

### 输出：
- 完整的测试架构
- 高覆盖率的测试用例
- 稳定的测试运行结果

---

## 阶段八：验证与优化 (Agent 8 - 质量保证师)

### 任务：
1. 执行完整的测试套件
2. 验证所有功能正常
3. 修复重构过程中的问题
4. 优化应用性能
5. 生成重构报告

### 输出：
- 测试报告
- 性能优化建议
- 架构重构完成报告

# 备份当前状态
cp -r src src.backup.$(date +%Y%m%d%H%M%S)

## 风险评估与缓解

### 高风险项目
1. **路由重构** - 可能影响所有页面访问
   - 缓解：分阶段迁移，保留备份
2. **API 客户端重构** - 可能影响数据通信
   - 缓解：保持接口兼容性，充分测试
3. **状态管理重构** - 可能导致数据丢失
   - 缓解：使用状态快照，测试状态恢复

### 中风险项目
1. **组件重构** - 可能影响 UI 显示
2. **样式重构** - 可能导致样式不一致
3. **测试重构** - 可能影响测试覆盖率

### 低风险项目
1. **目录结构调整** - 影响较小
2. **文件重命名** - 可通过工具自动化
