## 快速开始

### 环境要求

- **Node.js**: >= 18.0.0
- **npm**: >= 9.0.0
- **Git**: >= 2.30.0

### 安装依赖

```bash
# 克隆项目
git clone <repository-url>
cd strategic-task-management

# 安装依赖
npm install

# 或使用 pnpm（推荐）
pnpm install
```

### 启动开发服务器

```bash
# 启动开发服务器
npm run dev

# 服务器将运行在 http://localhost:3500
```

### 配置环境变量

创建 `.env.development` 文件：

```bash
# API 配置
VITE_API_TARGET=http://localhost:8080
VITE_USE_MOCK=false

# 开发服务器配置
VITE_DEV_SERVER_PORT=3500
VITE_DEV_AUTO_OPEN=true
```

## 开发工作流

### Git 工作流

```bash
# 1. 创建功能分支
git checkout -b feature/your-feature-name

# 2. 开发并提交
git add .
git commit -m "feat: add your feature"

# 3. 推送到远程
git push origin feature/your-feature-name

# 4. 创建 Pull Request
# 在 GitHub 上创建 PR
```

### 提交信息规范

使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型 (type)**:

- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具链更新

**示例**:

```bash
git commit -m "feat(indicator): add indicator fill form"
git commit -m "fix(auth): resolve token refresh issue"
git commit -m "docs: update API documentation"
```

### 代码审查清单

- [ ] 代码符合 ESLint 规范
- [ ] 代码符合 Prettier 格式
- [ ] 通过 TypeScript 类型检查
- [ ] 添加必要的注释
- [ ] 更新相关文档
- [ ] 通过所有测试
- [ ] 提交信息符合规范

## 编码规范

### TypeScript 规范

```typescript
// ✅ 推荐：使用 interface 定义对象类型
interface User {
  id: string
  name: string
  email: string
}

// ✅ 推荐：使用 type 定义联合类型
type Status = 'pending' | 'approved' | 'rejected'

// ✅ 推荐：使用 enum 定义枚举
enum UserRole {
  ADMIN = 'admin',
  USER = 'user',
  GUEST = 'guest'
}

// ✅ 推荐：函数参数使用类型注解
function getUserById(id: string): Promise<User> {
  return apiClient.get(`/users/${id}`)
}

// ❌ 不推荐：使用 any
function processData(data: any): any {
  // ...
}
```

### Vue 3 规范

```vue
<script setup lang="ts">
// 1. 导入
import { ref, computed, onMounted } from 'vue'

// 2. Props 定义
interface Props {
  title: string
  disabled?: boolean
}
const props = withDefaults(defineProps<Props>(), {
  disabled: false
})

// 3. Emits 定义
interface Emits {
  (e: 'update:modelValue', value: string): void
  (e: 'change', value: string): void
}
const emit = defineEmits<Emits>()

// 4. 响应式数据
const localValue = ref('')

// 5. 计算属性
const displayValue = computed(() => {
  return props.title.toUpperCase()
})

// 6. 方法
const handleChange = () => {
  emit('update:modelValue', localValue.value)
}

// 7. 生命周期
onMounted(() => {
  // 初始化
})
</script>

<template>
  <div class="component">
    {{ displayValue }}
  </div>
</template>

<style scoped lang="scss">
.component {
  // 样式
}
</style>
```

### 命名规范

## 测试

### 单元测试

使用 Vitest 进行单元测试：

```typescript
// tests/unit/utils/dateUtils.test.ts
import { describe, it, expect } from 'vitest'
import { formatDate } from '@/utils/dateUtils'

describe('formatDate', () => {
  it('should format date correctly', () => {
    const date = new Date('2026-03-15')
    const formatted = formatDate(date, 'YYYY-MM-DD')
    expect(formatted).toBe('2026-03-15')
  })
})
```

### 组件测试

```typescript
// tests/unit/components/Button.test.ts
import { describe, it, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import Button from '@/components/Button.vue'

describe('Button', () => {
  it('renders properly', () => {
    const wrapper = mount(Button, {
      props: {
        title: 'Click me'
      }
    })
    expect(wrapper.text()).toContain('Click me')
  })

  it('emits click event', async () => {
    const wrapper = mount(Button)
    await wrapper.trigger('click')
    expect(wrapper.emitted('click')).toBeTruthy()
  })
})
```

## 常见问题

### 1. 端口被占用

```bash
# 修改开发服务器端口
# vite.config.ts
server: {
  port: 3501  // 改为其他端口
}

# 或通过环境变量
VITE_DEV_SERVER_PORT=3501 npm run dev
```

### 2. API 跨域问题

开发环境使用代理：

```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

### 3. 类型错误

```bash
# 重新生成类型
npm run type-check

# 清理缓存
rm -rf node_modules/.vite
rm -rf node_modules/.cache
```

### 4. 样式不生效

检查：

- 是否使用了 `scoped`
- 是否有优先级更高的样式
- 是否使用了 CSS Modules

```vue
<style scoped lang="scss">
// 使用 :deep() 修改子组件样式
:deep(.child-element) {
  color: red;
}
</style>
```

### 5. 组件懒加载失败

```typescript
// 添加错误处理
const Component = defineAsyncComponent({
  loader: () => import('./Component.vue'),
  loadingComponent: LoadingSpinner,
  errorComponent: ErrorMessage,
  delay: 200,
  timeout: 10000
})
```

### 6. 路由跳转不刷新

```typescript
import { useRouter } from 'vue-router'

const router = useRouter()

// 使用 replace 避免历史记录堆叠
router.replace('/new-route')

// 或强制刷新
router.go(0)
```
